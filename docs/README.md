# docs — 设计文档与规范

## 内容

| 子目录 / 文件 | 说明 |
|---|---|
| `spec/` | 硬件架构规格文档 |
| `requirements/` | 软硬件需求拆分清单 |
| `methodology/` | VibeCoding 工作方法论与踩坑记录 |
| `meeting/` | 设计评审与会议纪要 |

## 职责

- 行业典型模型结构分析，输出硬件规格需求
- 维护 Software Feature 开发清单
- 维护仿真系统开发清单
- 记录 VibeCoding 实践过程中的经验教训

## 里程碑对应

- M1: 完成典型模型分析 → 输出 `spec/` 与 `requirements/` 初版
- M2: 并行开发中动态调整需求文档
